Section 1 — What you build (concrete)

By the end you run two apps + one API: a FastAPI backend that builds a vector index from your PDF (embeddings run locally with a small free model), a React frontend to upload and chat, and Groq for the LLM answers — fast Llama / Mixtral-class models with a free Groq API key (you create one account, no credit card for the free tier).

Section 2 — Every piece explained (not abstract)

PieceWhat it does in YOUR project
ReactRenders buttons. Sends the PDF as multipart/form-data to /upload. Sends your question as JSON to /chat. Shows response.data.answer on screen.
FastAPIPython web server. /upload reads bytes, extracts text, splits into chunks, builds FAISS with local Hugging Face embeddings (no key). /chat searches FAISS, builds a prompt, calls Groq for the LLM reply.
FAISSLibrary that stores embedding vectors (lists of floats) and finds nearest neighbours for your question’s embedding. That is your semantic search database for this bootcamp.
Embeddingssentence-transformers/all-MiniLM-L6-v2 runs on your laptop (downloads once). It turns each chunk into a vector so FAISS can retrieve similar text — no embedding API bill.
Gen AI (LLM)Default on Groq: llama-3.1-8b-instant (see Supported Models). Same API key, other ids like llama-3.3-70b-versatile or openai/gpt-oss-20b if your account can access them. With RAG, answers stay tied to your PDF. Change via env GROQ_MODEL.
SQLite (optional)Not required for the core demo. Use it later to store rows like filename, uploaded_at while FAISS still holds vectors — two layers: SQL for metadata, FAISS for meaning.

Tokens (real numbers): A model does not read “words” — it reads tokens (short pieces of text). A common English thumb rule is about 100 tokens for every 75 words — that is the “≈ ¾ of a word per token” idea: same paragraph → token count a bit higher than word count.

  • Example 1 — simple sentence (~11 words): “The college lab runs Python and FastAPI every week.” → often about 12–14 tokens (many small words = roughly one token each).
  • Example 2 — one long word splits: “Counterintuitive” → often 3–5 tokens (e.g. pieces like Counter, int, uitive) — here one “word” uses several tokens.
  • Example 3 — the ¾ rule in bulk: a 75-word paragraph is often near 100 tokens; a 150-word page section → often ~200 tokens (order of magnitude).

Your bootcamp splitter uses characters (e.g. 900 characters per chunk) for simplicity — that is usually ~200–250 tokens. Production RAG often limits by token count so chunk size stays predictable for the LLM context window.

Why two POST routes? /upload is slow (disk + PDF parse + embed many chunks). /chat is faster (one embed for the question + small vector search + one LLM call). Splitting keeps the UI simple: “index first, then chat.”

Section 3 — Install & run (copy / paste)

Create a free Groq API key first (steps below — no credit card needed to sign up and generate a key for normal class practice). Then install Python packages. After pip install, run the verify commands. First PDF upload may be slow while sentence-transformers downloads the small embedding model.

Free for students in this bootcamp: Groq gives developers free API access with rate limits (requests/tokens per minute), not “pay per click” from your bank. You do not enter card details just to create an account and an API key. If you later turn on paid billing in the console, that is optional — skip it for class. Official model IDs, speeds, and limits: console.groq.com/docs/models.

A. Create account (one time)

  1. Open groq.com → click Sign up / Get Started (wording may vary).
  2. Sign up with email or Google / GitHub if shown — complete any email verification link Groq sends you.
  3. Log in — you should reach the Groq Console (dark developer UI).

B. Create the API key (token)

  1. In the browser, open console.groq.com/keys (shortcut: Groq Console → left menu → API Keys).
  2. Click Create API key (or similar).
  3. Name it something you remember, e.g. soseeks-bootcamp → confirm.
  4. Groq shows the key once — copy the whole string. It always starts with gsk_.
  5. Store it in a password manager or notes privately. If you lose it, delete the old key in the console and create a new one (free).

C. Safety rules (same as real jobs)

  • Put the key only in backend env vars or a local .env file — never in React source, never in screenshots, never on GitHub.
  • If a key leaks, revoke it in the same Keys page and create a new one.

D. Which model ID to use?

  • Default in our main.py: llama-3.1-8b-instant — listed under production models on Supported Models, very fast for demos.
  • Optional upgrade: llama-3.3-70b-versatile or openai/gpt-oss-20b — check the table for context length and limits; heavier models use more of your per-minute allowance.
  • Change model without new key: set env GROQ_MODEL to any current model id from that docs page.

Section 4 — Backend: full main.py you can run

Save as backend/main.py. Embeddings use HuggingFaceEmbeddings on your laptop (no API). Chat uses ChatGroq with your GROQ_API_KEY — same pattern as Groq’s OpenAI-compatible endpoint. If an import fails, re-run pip install from Section 3.

Before uvicorn: set GROQ_API_KEY in the same terminal (free key from console.groq.com/keys). Never commit the key to Git.
backend / main.py
import os
import tempfile
from typing import Optional

from fastapi import FastAPI, UploadFile, File, HTTPException
from fastapi.middleware.cors import CORSMiddleware
from langchain_community.document_loaders import PyPDFLoader
from langchain_community.embeddings import HuggingFaceEmbeddings
from langchain_community.vectorstores import FAISS
from langchain_groq import ChatGroq
from langchain_text_splitters import RecursiveCharacterTextSplitter
from langchain_core.messages import SystemMessage, HumanMessage

# Groq chat model — change with env GROQ_MODEL (see https://console.groq.com/docs/models )
GROQ_MODEL = os.getenv("GROQ_MODEL", "llama-3.1-8b-instant")
# Local embeddings (first run downloads ~90MB model to Hugging Face cache)
HF_EMBED_MODEL = "sentence-transformers/all-MiniLM-L6-v2"

app = FastAPI(title="Soseeks RAG Bootcamp (Groq + local embeddings)")

app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:3000"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

vectorstore: Optional[FAISS] = None

BASE = os.path.dirname(os.path.abspath(__file__))
DATA_DIR = os.path.join(BASE, "data", "raw")
os.makedirs(DATA_DIR, exist_ok=True)


def require_groq_key():
    if not os.getenv("GROQ_API_KEY"):
        raise HTTPException(
            500,
            "GROQ_API_KEY is not set. Create a free key at https://console.groq.com/keys",
        )


@app.get("/health")
def health():
    return {
        "status": "ok",
        "has_index": vectorstore is not None,
        "groq_model": GROQ_MODEL,
        "embed_model": HF_EMBED_MODEL,
    }


@app.post("/upload")
async def upload(file: UploadFile = File(...)):
    global vectorstore
    if not file.filename.lower().endswith(".pdf"):
        raise HTTPException(400, "Only .pdf files are supported in this bootcamp version.")

    raw = await file.read()
    disk_path = os.path.join(DATA_DIR, file.filename)
    with open(disk_path, "wb") as f:
        f.write(raw)

    fd, tmp_path = tempfile.mkstemp(suffix=".pdf")
    os.write(fd, raw)
    os.close(fd)
    try:
        loader = PyPDFLoader(tmp_path)
        documents = loader.load()
    finally:
        os.remove(tmp_path)

    splitter = RecursiveCharacterTextSplitter(chunk_size=900, chunk_overlap=120)
    chunks = splitter.split_documents(documents)

    embeddings = HuggingFaceEmbeddings(model_name=HF_EMBED_MODEL)
    vectorstore = FAISS.from_documents(chunks, embeddings)

    return {
        "ok": True,
        "filename": file.filename,
        "pages": len(documents),
        "chunks": len(chunks),
    }


@app.post("/chat")
async def chat(payload: dict):
    global vectorstore
    require_groq_key()
    question = (payload.get("question") or "").strip()
    if not question:
        raise HTTPException(400, 'JSON body must include "question".')
    if vectorstore is None:
        raise HTTPException(400, "Upload a PDF first using POST /upload.")

    docs_found = vectorstore.similarity_search(question, k=5)
    context = "\n\n---\n\n".join(d.page_content for d in docs_found)

    system = (
        "You are a helpful tutor. Answer ONLY using the CONTEXT blocks. "
        "If the answer is not in the context, say you do not have that information in the uploaded file."
    )
    human = f"CONTEXT:\n{context}\n\nQUESTION:\n{question}"

    llm = ChatGroq(model=GROQ_MODEL, temperature=0)
    reply = llm.invoke(
        [SystemMessage(content=system), HumanMessage(content=human)]
    )

    return {
        "answer": reply.content,
        "chunks_used": len(docs_found),
    }

Section 5 — Frontend: upload + chat (App.js)

Replace frontend/src/App.js with this. It uses a hidden file input, FormData for /upload, and JSON for /chat. Status text tells the student each step succeeded.

frontend / src / App.js
import { useRef, useState } from "react";
import axios from "axios";
import "./App.css";

const API = "http://127.0.0.1:8000";

export default function App() {
  const fileRef = useRef(null);
  const [question, setQuestion] = useState("");
  const [answer, setAnswer] = useState("");
  const [status, setStatus] = useState("Upload a PDF, then ask a question.");

  async function uploadPdf() {
    const f = fileRef.current?.files?.[0];
    if (!f) {
      setStatus("Choose a .pdf file first.");
      return;
    }
    const fd = new FormData();
    fd.append("file", f);
    setStatus("Uploading and building index…");
    const res = await axios.post(`${API}/upload`, fd, {
      headers: { "Content-Type": "multipart/form-data" },
    });
    setStatus(
      `Indexed: ${res.data.filename} — ${res.data.chunks} chunks from ${res.data.pages} pages.`
    );
  }

  async function ask() {
    if (!question.trim()) return;
    setStatus("Asking the model…");
    const res = await axios.post(`${API}/chat`, { question });
    setAnswer(res.data.answer);
    setStatus(`Used ${res.data.chunks_used} text chunks from your PDF.`);
  }

  return (
    <div className="App" style={{ maxWidth: 720, margin: "40px auto", padding: 16 }}>
      <h1>My RAG Chat (Soseeks Bootcamp)</h1>

      <section style={{ marginBottom: 24 }}>
        <h2>1) Upload PDF</h2>
        <input ref={fileRef} type="file" accept="application/pdf" />
        <button type="button" onClick={uploadPdf} style={{ marginLeft: 12 }}>
          Upload to server
        </button>
      </section>

      <section style={{ marginBottom: 24 }}>
        <h2>2) Chat with your file</h2>
        <textarea
          rows={3}
          style={{ width: "100%" }}
          placeholder="Ask something that is answered inside the PDF…"
          value={question}
          onChange={(e) => setQuestion(e.target.value)}
        />
        <br />
        <button type="button" onClick={ask} style={{ marginTop: 8 }}>
          Send question
        </button>
      </section>

      <p><strong>Status:</strong> {status}</p>
      <h3>Answer</h3>
      <pre
        style={{
          whiteSpace: "pre-wrap",
          background: "#111",
          color: "#e2e8f0",
          padding: 16,
          borderRadius: 8,
        }}
      >
        {answer || "(no answer yet)"}
      </pre>
    </div>
  );
}
CORS: If the browser console shows a CORS error, confirm allow_origins in main.py includes exactly your React URL (usually http://localhost:3000) and that you restarted uvicorn after edits.

Section 6 — Exact request / response flow

StepWhoHTTPBody
1ReactPOST /uploadmultipart/form-data field name file (the PDF binary)
2FastAPI—Saves PDF under data/raw/, loads pages, splits, builds vectors with HuggingFaceEmbeddings locally, builds FAISS in RAM.
3ReactPOST /chat{"question":"..."} (JSON)
4FastAPI—Embeds question locally → FAISS top-5 chunks → builds prompt → ChatGroq (e.g. llama-3.1-8b-instant) → returns {"answer":"...","chunks_used":5}
5React—Renders res.data.answer in the grey box.

That is the full stack + Gen AI + vector “database” loop: no magic — every hop is a function you can log with print() in Python and console.log() in React.
